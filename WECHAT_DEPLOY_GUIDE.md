# 微信静态网页托管 + 微信云数据库 完整部署指南

## 一、前期准备

### 1.1 注册微信云开发账号
1. 访问 [微信云开发控制台](https://console.cloud.tencent.com/tcb)
2. 使用微信扫码登录
3. 创建一个新的云开发环境

### 1.2 获取环境ID
1. 在云开发控制台中，进入「环境」→ 「环境设置」
2. 复制环境ID（格式类似：cloudbase-xxxxx）
3. 更新 `cloudbase.config.js` 中的 `envId` 配置
4. 更新 `src/utils/cloud.js` 中的环境ID（第19行）

## 二、数据库配置

### 2.1 创建数据库集合
在云开发控制台中创建以下集合：

1. **users** - 用户表
2. **venues** - 场馆表
3. **climb_requests** - 约爬请求表
4. **climb_request_participants** - 约爬参与者表
5. **posts** - 帖子表
6. **chat_conversations** - 聊天会话表
7. **chat_conversation_members** - 会话成员表
8. **chat_messages** - 聊天消息表

### 2.2 设置数据库权限
为每个集合设置合适的权限规则：

```json
// users 集合权限 - 仅创建者可读写
{
  "read": "doc._id == auth.openid",
  "write": "doc._id == auth.openid"
}

// venues 集合权限 - 所有人可读，仅管理员可写
{
  "read": true,
  "write": false
}

// 其他集合根据业务需求设置权限
```

## 三、云存储配置

1. 在云开发控制台中，进入「存储」
2. 创建以下文件夹：
   - `/avatars` - 用户头像
   - `/posts` - 帖子图片
   - `/uploads` - 通用上传

## 四、构建项目

### 4.1 安装依赖
```bash
npm install
```

### 4.2 构建H5版本
```bash
# 两种方式都可以：
npm run build:h5
# 或者使用简写命令：
npm run build
```

构建完成后，会在 `dist/build/h5` 目录下生成静态文件。

## 五、部署静态网站

### 5.1 开启静态网站托管
1. 在云开发控制台中，进入「静态网站托管」
2. 点击「开通」，按照提示完成开通

### 5.2 上传静态文件
1. 在「静态网站托管」页面，点击「文件管理」
2. 将 `dist/build/h5` 目录下的所有文件上传
3. 确保 `index.html` 在根目录下

### 5.3 配置自定义域名（可选）
1. 在「静态网站托管」→「设置」→「域名管理」
2. 添加自定义域名
3. 按照提示完成DNS解析和SSL证书配置

## 六、安全配置

### 6.1 配置登录方式
1. 在云开发控制台中，进入「环境」→「登录设置」
2. 开启「匿名登录」或「微信网页授权登录」

### 6.2 配置Web安全域名
1. 在云开发控制台中，进入「环境」→「安全配置」
2. 添加你的网站域名到「Web安全域名」列表中

## 七、测试验证

### 7.1 访问网站
1. 在「静态网站托管」页面找到默认域名
2. 在浏览器中访问该域名
3. 测试各项功能是否正常

### 7.2 检查数据库连接
1. 测试用户注册/登录功能
2. 检查数据是否能正常写入数据库
3. 验证数据读取是否正常

## 八、常见问题

### Q: 云开发初始化失败怎么办？
A: 检查以下几点：
- 环境ID是否正确
- 网络连接是否正常
- 浏览器控制台是否有详细错误信息

### Q: 数据库权限配置错误？
A: 在云开发控制台重新设置集合权限规则，确保应用有足够的读写权限。

### Q: 静态网站无法访问？
A: 确认：
- 静态文件是否全部上传成功
- 域名配置是否正确
- 是否有防火墙或安全策略阻止访问

## 九、后续优化

1. **性能优化**：启用CDN加速，压缩静态资源
2. **安全加固**：设置更严格的数据库权限规则
3. **监控告警**：配置云函数和数据库的监控告警
4. **备份恢复**：定期备份数据库数据

---

**注意**：部署前请确保已备份重要数据，并在测试环境充分验证后再上线生产环境。
